Automatic part-of-speech tagging for Swedish via JSON

Stagger finds detailed Part-of-speech tags (Noun, Verb, ...) and Named-entities (Person, Location, Institution) from Swedish sentences. This API makes Stagger available via a JSON API.

Pros and Cons

Stagger does a lot of work for you:

  • Tokenization - Split your text into words, handling tricky cases like abbreviations.
  • Sentence splitting - Split your tokens into sentences, handling periods in strange places.
  • Lemmatisation - Find the base form for each word. Very useful for things like search indexing.
  • Part-of-speech tagging - Find what category a specific word belongs to, including specifics like singular/plural and passive/active forms.
  • Named-entity Recognition - Based on the POS tags, find which words are actually people, places, and institutions.

JSON Tagger (this project) makes Stagger easier to use:

  • Adds a JSON API. No need to parse tab separated strings on stdout.
  • No need to run and deploy Java yourself. JSON Tagger is hosted, so you don't need to setup anything yourself.
  • Separates tags from entites, to make them easier to use separately. Also merges multi-token entities to make them even easier to parse.

General things to keep in mind:

  • The Swedish language model that comes with Stagger is trained on editorial text. This means parsing editorial content (news articles, product descriptions, ...) with the highest accuracy. If you need to parse social content, you need to train your own model and run Stagger yourself.
  • The Part-of-speech tagger is very accurate (96.6%).
  • The Named-entity Recognition engine isn't as accurate (70.7%). The lower accuracy is because NER is a much harder problem. 70-80% is the industry standard.
